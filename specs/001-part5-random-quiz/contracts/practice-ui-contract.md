# Practice UI Contract

## Purpose

ブラウザ上の TOEIC Part 5 練習画面が満たすべき入出力と状態遷移を定義する。

## Screen Contract

### Initial Load

- 入力: 練習画面を開く。
- 表示要件:
  - 1 問分の問題文を表示する。
  - 4 つの選択肢を表示する。
  - 主要操作として回答操作だけを目立たせる。
  - 読み込み中はローディング状態を表示する。

### Answer Submission

- 入力: 学習者が 1 つの選択肢を選び、解答を確定する。
- システム応答:
  - 同じ問題への再解答を防止する。
  - 正誤を即時表示する。
  - 正答を表示する。
  - 解説を表示する。
  - 次の問題へ進む操作を表示する。

### Next Question Transition

- 入力: 学習者が次の問題へ進む。
- システム応答:
  - 新しい問題を表示する。
  - 前問の選択状態やフィードバック表示を引き継がない。
  - 次問表示前に必要ならローディング状態を表示する。

## Data Contract

各問題は以下の情報を持たなければならない。

| Field | Required | Description |
|-------|----------|-------------|
| `id` | Yes | 問題識別子 |
| `prompt` | Yes | 問題文 |
| `choices` | Yes | 4 件の選択肢 |
| `correctChoiceId` | Yes | 正答選択肢 ID |
| `explanation` | Yes | 解説本文 |

## Error Contract

- 問題データの読み込みに失敗した場合は、空白画面にせず再試行可能なエラー表示を行う。
- 問題データの形式が不正な場合は、その問題を出題対象から除外するか、練習開始を止めて明示的なエラーを表示する。
- 不正な入力状態で解答処理が呼ばれた場合は、画面状態を破壊せず、学習者に再操作を促す。

## Mobile Contract

- 主要要素は横スクロールなしで閲覧・操作できること。
- 選択肢と次へ進む操作はタップしやすいサイズと間隔を持つこと。
- 正誤表示と解説表示はモバイル画面で読み切れる順序で並ぶこと。
